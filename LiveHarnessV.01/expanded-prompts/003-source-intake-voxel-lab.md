# Product Requirements

Create a browser playable voxel world with procedural terrain, HUD, inventory bar, input test surface, debug state surface, and domain shaped state for movement, inventory, world updates, and sequence objectives.

Before the build swarm runs, the harness should run source intake loops for NexusRealtime core, NexusRealtime ProtoKits, and the local KitBuilder03 project, fuse those capability maps, and use the resulting integration plan as reference and validation context.

The public game should expose window.GameHost.getState() with player, world, inventory, domainTrace, events, input, sequence, renderer diagnostics, and a compact repo capability reference file kept out of the HUD.
